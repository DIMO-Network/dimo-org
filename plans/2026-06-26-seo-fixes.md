# DIMO.org SEO Fixes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise the dimo.org SEO health score from ~48 to ~80+ by fixing the
audit's Critical/High/Medium findings: performance (24 MB GIFs), trust (legal
pages + fabricated review schema), crawl/index hygiene, and the AI-search signal
layer.

**Architecture:** This is a Docusaurus 3.9.2 site on Vercel behind Cloudflare.
Most fixes are edits to React page components (`src/pages/*.tsx`), site config
(`docusaurus.config.ts`, `vercel.json`), static assets (`static/`), and
blog/docs markdown. There is no unit-test suite for SEO; each task's "test" is a
build (`npm run build`), a type/lint check (`npm run check-all`), and/or a
`curl`/`grep` assertion against the built output or live site.

**Tech Stack:** Docusaurus 3.9.2, React 19, TypeScript, Node ≥24, Vercel,
Cloudflare, Algolia DocSearch, ffmpeg + cwebp (for asset conversion — see Global
Constraints).

## Global Constraints

- **Canonical host:** `https://www.dimo.org` (www + https). All absolute URLs
  use this host. Verbatim.
- **Prettier:** 80-char width, single quotes, trailing commas (es5), 2-space
  indent. MDX uses `proseWrap: preserve`, 120-char width. Run `npm run format`
  after edits.
- **ESLint:** zero-warning policy, max line length 80. Run `npm run lint`.
- **Verification gate per task:** `npm run check-all` (typecheck + lint +
  format:check) must pass, and `npm run build` must succeed, before any commit.
- **Asset tooling:** GIF→video and PNG→WebP tasks require `ffmpeg` and `cwebp`
  (libwebp). Neither is on the dev machine — install first:
  `brew install ffmpeg webp`. If unavailable, do those conversions in CI or skip
  to the JSX/markup half and land the binary separately.
- **Do NOT save plans, scratch, or reports under `docs/`, `blog/`, `src/pages/`,
  or `static/`** — Docusaurus publishes those. This plan lives in `plans/` (not
  served).
- **No `aggregateRating` / `Review` schema** may be added back unless it
  reflects real, on-page, individually-attributed reviews.
- **Schema deprecations:** never add `HowTo` (deprecated 2023). `FAQPage` gives
  no Google SERP feature (retired 2026-05-07) — only use `QAPage` for genuine
  Q&A, and only for AI-citation value.
